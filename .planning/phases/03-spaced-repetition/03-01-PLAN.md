<feature>
  <name>SM-2 Spaced Repetition Algorithm</name>
  <files>backend/internal/srs/sm2.go, backend/internal/srs/sm2_test.go</files>
  <behavior>
    SM-2 algorithm with the following behavior:

    Types:
    - SM2Input: Score (int, 1-5), EaseFactor (float64), IntervalDays (int), RepetitionCount (int)
    - SM2Output: EaseFactor (float64), IntervalDays (int), RepetitionCount (int), NextReview (time.Time)

    Function: Calculate(input SM2Input, now time.Time) SM2Output

    Interval rules:
    - Score >= 3, repetition 0 (first success): interval = 1 day
    - Score >= 3, repetition 1 (second success): interval = 6 days
    - Score >= 3, repetition >= 2: interval = round(previous_interval * ease_factor)
    - Score < 3 (failed): interval = 1 day, repetition resets to 0

    Ease factor rules:
    - Formula: EF' = EF + (0.1 - (5 - score) * (0.08 + (5 - score) * 0.02))
    - Floor: 1.3 (never drops below, per SRS-04)
    - Applied AFTER interval calculation
    - When score < 3: ease factor still updated but floored at 1.3

    NextReview: now.Add(interval_days * 24h)

    Test cases:
    - First review, score 5: interval=1, EF increases from 2.5 to 2.6
    - First review, score 4: interval=1, EF stays at 2.5 (0.1 - 0.1 = 0)
    - First review, score 3: interval=1, EF decreases to 2.36
    - Second review, score 5: interval=6, EF increases
    - Third review, score 5 with EF=2.5: interval=round(6*2.5)=15
    - Score 2 (fail): interval=1, repetition=0, EF decreased but >= 1.3
    - Score 1 (fail): interval=1, repetition=0, EF floored at 1.3
    - Ease factor at 1.3, score 1: EF remains 1.3 (floor enforced)
    - Large interval (30 days), score 5: interval grows correctly
    - Default initial state (EF=2.5, interval=0, reps=0): first review works
  </behavior>
  <implementation>
    Create package `backend/internal/srs/` with:

    1. `sm2.go`: Define SM2Input and SM2Output structs. Implement Calculate as a pure function.
       - Use math.Round for interval calculation
       - Use math.Max for EF floor enforcement
       - All time calculations use time.Duration

    2. `sm2_test.go`: Table-driven tests covering all cases above.
       - Use a fixed `now` time for deterministic NextReview assertions
       - Test edge cases: minimum EF, maximum score, zero interval
  </implementation>
</feature>
