**Gathered:** 2026-04-03

<domain>
## Phase Boundary

Implement SM-2 spaced repetition algorithm server-side. Sessions produce scores that feed scheduling. Student home screen shows "Dags att repetera" cards for topics past their review date. Covers SRS-01..04.

</domain>

<decisions>
## Implementation Decisions

### Review dashboard
- Cards show: ämne + ämnesområde + "X dagar sedan" + "Repetera" button
- Sorted by most overdue first (biggest gap between now and next_review)
- Empty state: encouraging message "Inget att repetera just nu! Prova en ny övning." with link to subject selection
- Cards displayed on student home (/study)

### SM-2 tuning
- Claude's Discretion: ease factor floor (1.3 from spec), initial interval, overdue item handling
- Standard SM-2 algorithm — no need for SM-2+ variant
- Score comes from AI assessment in Phase 2 (already implemented)

### Review flow
- Clicking a review card starts a new session on that topic's exercise (same as starting any exercise)
- No special "review mode" — the spaced repetition just determines WHICH exercises to surface

### Claude's Discretion
- SM-2 algorithm parameters (initial interval, graduating interval)
- Overdue item handling strategy
- Card visual styling details
- Maximum number of review cards shown at once

</decisions>
