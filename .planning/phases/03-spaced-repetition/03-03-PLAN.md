  <action>
    **1. Create `backend/internal/srs/store.go`:**

    Define SRSStore interface following the ChatStore pattern:
    ```go
    type SRSStore interface {
        UpsertReviewSchedule(ctx context.Context, arg queries.UpsertReviewScheduleParams) (queries.ReviewSchedule, error)
        ListDueReviews(ctx context.Context, studentID pgtype.UUID) ([]queries.ListDueReviewsRow, error)
        GetReviewSchedule(ctx context.Context, arg queries.GetReviewScheduleParams) (queries.ReviewSchedule, error)
    }
    ```

    Implement QueriesStore adapter wrapping *queries.Queries (same pattern as chat/store.go).

    **2. Create `backend/internal/srs/handler.go`:**

    SRSHandler struct with store field. Constructor NewSRSHandler(store SRSStore).

    `ListDueReviews` handler for `GET /api/reviews/due`:
    - Get studentID from JWT context via auth.GetUserIDFromContext
    - Call store.ListDueReviews with student UUID
    - Return JSON array with fields: id, exerciseId, exerciseTitle, topicName, topicSlug, subjectName, subjectSlug, nextReview, daysOverdue
    - Calculate daysOverdue as int(time.Since(nextReview).Hours() / 24)
    - Use the same writeJSON and uuidToString helpers (import from chat package or duplicate — prefer duplicating the 2 small helpers to avoid circular imports)

    **3. Add SRS hook to `backend/internal/chat/handler.go`:**

    Add a new method to ChatHandler or accept an SRS callback. The cleanest approach:
    - Add an `onSessionEnd` callback field to ChatHandler: `onSessionEnd func(ctx context.Context, studentID, exerciseID pgtype.UUID, score int)`
    - In EndSession, after successfully ending the session and before writing the response, call the callback if set
    - The callback should NOT block the response — if it fails, log the error and continue

    Alternatively (simpler): Add UpsertReviewSchedule and GetReviewSchedule to ChatStore interface, and do the SM-2 calculation inline in EndSession. This keeps everything in one package.

    **Preferred approach (simpler, follows existing patterns):**
    - Add `UpsertReviewSchedule` and `GetReviewSchedule` methods to ChatStore interface
    - Add implementations to chat/QueriesStore
    - In EndSession, after score is saved:
      1. Get existing review schedule (if any) via GetReviewSchedule
      2. Build SM2Input from existing data (or defaults if first review)
      3. Call srs.Calculate(input, time.Now())
      4. Call UpsertReviewSchedule with SM2Output values
      5. If any step fails, log.Error and continue (don't fail the EndSession response)

    Import `srs` package in chat/handler.go for the Calculate function.

    **4. Update `backend/internal/chat/store.go`:**

    Add to ChatStore interface:
    ```go
    UpsertReviewSchedule(ctx context.Context, arg queries.UpsertReviewScheduleParams) (queries.ReviewSchedule, error)
    GetReviewSchedule(ctx context.Context, arg queries.GetReviewScheduleParams) (queries.ReviewSchedule, error)
    ```

    Add implementations to QueriesStore.
