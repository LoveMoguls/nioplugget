  <behavior>
    Invite:
    - GenerateInviteToken() returns a URL-safe string of at least 43 characters (256 bits)
    - Two calls produce different tokens
    - Token contains only URL-safe characters (base64url)

    Rate limiter:
    - NewPINRateLimiter(maxAttempts=5, lockoutDuration=15min) creates limiter
    - Check(studentID) on first attempt: returns (allowed=true, remaining=4)
    - Check(studentID) after 4 more fails: returns (allowed=false, remaining=0)
    - Check(studentID) during lockout: returns (allowed=false, remaining=0)
    - Check(studentID) after lockout expires: returns (allowed=true, remaining=4) — reset
    - Reset(studentID) clears attempts (called on successful login)
    - Different studentIDs have independent counters
  </behavior>
  <action>
    1. Create `internal/child/invite.go`:
       - `GenerateInviteToken() (string, error)` — 32 bytes from crypto/rand, base64.URLEncoding.EncodeToString

    2. Create `internal/child/ratelimit.go`:
       - `type PINRateLimiter struct` with sync.Mutex, map[string]*attempt
       - `type attempt struct { count int; lockedUntil time.Time }`
       - `NewPINRateLimiter(maxAttempts int, lockoutDuration time.Duration) *PINRateLimiter`
       - `Check(studentID string) (allowed bool, remaining int)`:
         - If locked and lockout not expired: return false, 0
         - If locked and lockout expired: reset count
         - Increment count
         - If count >= max: set lockedUntil = now + lockoutDuration, return false, 0
         - Return true, max - count
       - `Reset(studentID string)` — delete entry (called after successful PIN login)

    Write tests first (RED), implement (GREEN).
  </action>
  <verify>
    <automated>cd backend && go test ./internal/child/ -run "TestInvite|TestRateLimit" -v</automated>
  </verify>

<task type="auto" tdd="true">
  <name>Task 2: Child management and invite/activation handlers</name>
  <files>backend/internal/child/handler.go, backend/internal/child/handler_test.go</files>
  <behavior>
    Create child:
    - POST /api/children with {name} creates student with invite link, returns 201 with {id, name, inviteURL}
    - POST /api/children with empty name returns 400

    Generate invite:
    - POST /api/children/{id}/invite generates new invite for existing child, returns 200 with {inviteURL, expiresAt}
    - POST /api/children/{id}/invite for another parent's child returns 403

    List children:
    - GET /api/children returns parent's children with status (activated/pending)

    Activate invite:
    - POST /api/invite/{token}/activate with {name, pin} activates account, returns 200 + child JWT cookie
    - PIN must be exactly 4 digits, otherwise 400
    - Expired token returns 410 {"error": "Lanken har gatt ut. Be din foralder skapa en ny."}
    - Already-used token returns 410 (same message)
    - Activation atomically: sets pin_hash, activated_at, clears invite_code in single UPDATE

    Child login:
    - POST /api/child/login with {parentEmail, studentName, pin} returns 200 + child JWT cookie
    - Wrong PIN returns 401 with remaining attempts: {"error": "Fel PIN-kod", "remaining": N}
    - Locked account returns 429 {"error": "For manga forsok. Forsoek igen om 15 minuter."}
    - Successful login resets rate limiter
    - GET /api/child/names?parent_email={email} returns list of activated child names for that parent (public, for the name-selection UI)
  </behavior>
  <action>
    Create `internal/child/handler.go`:
    - `type ChildHandler struct { queries; rateLimiter *PINRateLimiter }`
    - `NewChildHandler(q, rl) *ChildHandler`

    - `Create(w, r)` — POST /api/children (parent-protected):
      - Extract parent_id from JWT
      - Decode JSON: {name}
      - Generate invite token, set expiry 72h from now
      - queries.CreateStudent(parent_id, name, invite_code, invite_expires_at)
      - Return 201 {"id": id, "name": name, "inviteURL": FRONTEND_URL + "/invite/" + token, "expiresAt": expiresAt}

    - `GenerateInvite(w, r)` — POST /api/children/{id}/invite (parent-protected):
      - Verify child belongs to authenticated parent
      - Generate new invite token, update student record
      - Return 200 with new invite URL and expiry

    - `List(w, r)` — GET /api/children (parent-protected):
      - queries.GetStudentsByParentID(parent_id)
      - Return 200 with array: [{id, name, activated: bool, createdAt}]

    - `Activate(w, r)` — POST /api/invite/{token}/activate (public):
      - Decode JSON: {name, pin}
      - Validate PIN: must be exactly 4 digits (regex: ^[0-9]{4}$), return 400 if not
      - Hash PIN with auth.HashPassword (same Argon2id as passwords)
      - queries.ActivateStudent(token, pin_hash) — atomic UPDATE with WHERE checks
      - If no row returned: token invalid/expired/used — return 410 {"error": "Lanken har gatt ut. Be din foralder skapa en ny."}
      - Generate child JWT (role: "child")
      - Set httpOnly cookie
      - Return 200 {"id": studentID, "name": name, "message": "Kontot ar aktiverat!"}

    - `PINLogin(w, r)` — POST /api/child/login (public):
      - Decode JSON: {parentEmail, studentName, pin}
      - Look up parent by email, then student by name + parent_id
      - If not found: return 401 {"error": "Fel PIN-kod"} (don't reveal which part is wrong)
      - Check rate limiter: if locked, return 429
      - Compare PIN hash
      - If wrong: return 401 {"error": "Fel PIN-kod", "remaining": N}
      - If correct: reset rate limiter, generate child JWT, set cookie
      - Return 200 {"id": studentID, "name": name}

    - `ListNames(w, r)` — GET /api/child/names?parent_email={email} (public):
      - Look up parent by email
      - queries.ListStudentNamesByParentID — only activated students
      - Return 200 [{name: string}]

    Wire all routes into main.go.
  </action>

  </verify>
  <done>Full child lifecycle works: parent creates child -> invite generated -> child activates with PIN -> child logs in with name + PIN. Rate limiting enforced. Invite atomically invalidated.</done>
