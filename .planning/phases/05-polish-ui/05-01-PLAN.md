Replace the existing achromatic color palette in app.css with calm blue/green tones using oklch format (already in use).

**Light mode palette guidance:**
- `--color-background`: near-white with very subtle warm tint (e.g., `oklch(0.99 0.002 210)`)
- `--color-foreground`: deep blue-gray (e.g., `oklch(0.20 0.02 240)`)
- `--color-primary`: muted teal/blue-green for buttons and links (e.g., `oklch(0.45 0.12 195)`)
- `--color-primary-foreground`: white/near-white for text on primary
- `--color-secondary`: very light blue-green tint for secondary surfaces
- `--color-muted`: light gray with subtle blue undertone
- `--color-muted-foreground`: medium gray-blue for secondary text
- `--color-accent`: soft sage/green for hover states and accents
- `--color-accent-foreground`: dark text for readability on accent
- `--color-destructive`: keep existing red (works across themes)
- `--color-border`: light gray with subtle blue tint
- `--color-input`: same as border
- `--color-ring`: medium blue-green for focus rings
- `--color-card`: white or near-white
- `--color-card-foreground`: same as foreground

**Dark mode palette (in `.dark` selector):**
- Mirror the light palette structure but with appropriate dark variants
- Background: deep blue-gray (e.g., `oklch(0.17 0.02 240)`)
- Foreground: light cream/white
- Primary: slightly brighter teal that works on dark backgrounds
- Maintain sufficient contrast ratios (WCAG AA: 4.5:1 for text)

Keep the existing `@theme`, `@custom-variant dark`, `@layer base` structure.
Keep `--radius-*` values unchanged.
Keep `--font-sans` unchanged (system fonts per CONTEXT discretion).

Do NOT add animations or transitions. Do NOT add gamification colors (gold, orange streak colors, etc.).
