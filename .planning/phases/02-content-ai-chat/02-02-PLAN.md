  <action>
Create `backend/internal/content/` package with:

**store.go** — ContentStore interface over sqlc queries (same pattern as apikey.QueriesStore):
```go
type ContentStore interface {
    ListSubjects(ctx context.Context) ([]queries.Subject, error)
    GetSubjectBySlug(ctx context.Context, slug string) (queries.Subject, error)
    ListTopicsBySubjectID(ctx context.Context, subjectID pgtype.UUID) ([]queries.Topic, error)
    GetTopicBySlug(ctx context.Context, subjectSlug, topicSlug string) (queries.GetTopicBySlugRow, error)
    ListExercisesByTopicID(ctx context.Context, topicID pgtype.UUID) ([]queries.ListExercisesByTopicIDRow, error)
}
```

Implement `QueriesStore` struct wrapping `*queries.Queries`.

**handler.go** — ContentHandler with three endpoints:
1. `GET /api/subjects` — list all subjects ordered by display_order. Returns JSON array `[{id, name, slug}]`.
2. `GET /api/subjects/{subjectSlug}/topics` — list topics for a subject. Look up subject by slug first, then list topics. Returns JSON `{subject: {name, slug}, topics: [{id, name, slug}]}`.
3. `GET /api/topics/{subjectSlug}/{topicSlug}/exercises` — list exercises for a topic. Look up topic by subject+topic slugs, then list exercises. Returns JSON `{subject: {name, slug}, topic: {name, slug}, exercises: [{id, title, description, difficulty_order}]}`. Note: system_prompt is NOT returned to the client.

Use `encoding/json` for responses. Return 404 if slug not found. Follow existing handler patterns (no framework — plain http.Handler with chi URL params).

**main.go** — Register content routes. These require child JWT (ChildOnly middleware):
```go
// Content routes (child only)
r.Route("/api/subjects", func(r chi.Router) {
    r.Use(jwtauth.Verifier(tokenAuth))
    r.Use(jwtauth.Authenticator(tokenAuth))
    r.Use(auth.ChildOnly)
    r.Get("/", contentHandler.ListSubjects)
    r.Get("/{subjectSlug}/topics", contentHandler.ListTopics)
})
r.Route("/api/topics", func(r chi.Router) {
    r.Use(jwtauth.Verifier(tokenAuth))
    r.Use(jwtauth.Authenticator(tokenAuth))
    r.Use(auth.ChildOnly)
    r.Get("/{subjectSlug}/{topicSlug}/exercises", contentHandler.ListExercises)
})
```

Initialize ContentStore and ContentHandler in main() following the existing pattern (apiKeyStore/apiKeyHandler).

Add content and session API functions to `frontend/src/lib/api.ts`:
```typescript
// Content
export const content = {
    subjects: () => apiFetch('/api/subjects'),
    topics: (subjectSlug: string) => apiFetch(`/api/subjects/${subjectSlug}/topics`),
    exercises: (subjectSlug: string, topicSlug: string) => apiFetch(`/api/topics/${subjectSlug}/${topicSlug}/exercises`),
};

// Sessions
export const sessions = {
    create: (exerciseId: string) => apiFetch('/api/sessions', { method: 'POST', body: JSON.stringify({ exerciseId }) }),
};
```
