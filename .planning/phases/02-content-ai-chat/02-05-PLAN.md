  <action>
Add `EndSession` method to ChatHandler:

`POST /api/sessions/{id}/end` — End a session with scoring:

Flow:
1. Extract session ID from URL, verify session belongs to authenticated student
2. Verify session is not already ended (ended_at IS NULL), return 400 if already ended
3. Load all messages for the session
4. Load the exercise (for system prompt context)
5. Look up student -> parent -> API key, decrypt
6. Call ScoreSession to get score + summary + feedback
7. Save summary as a final assistant message in the messages table (so it appears in chat history)
8. Update session: set ended_at, score, and summary via EndSession query
9. Return JSON response: `{score: int, summary: string, feedback: string, endedAt: string}`

Error handling:
- If API key not found: return 500 with "Foralderns API-nyckel saknas"
- If scoring fails: still end the session with score=3 and generic summary "Passet avslutades utan bedomning."
- If session not found or doesn't belong to student: return 404

Register the route in main.go if not already done (it should be part of the /api/sessions route group from Plan 03):
```go
r.Post("/{id}/end", chatHandler.EndSession)
```
